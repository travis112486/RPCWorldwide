# Implementation Plan — Ticket #61: Create and Manage Presentations

## Summary
Add a "Presentations" tab to the casting sub-navigation with a page for creating, listing, and managing shareable presentation links for external client review.

## Issue Requirements
- New "Presentations" tab in casting project sub-navigation
- "New Presentation" button → form: name, type (Live/Custom), optional password, optional expiry
- For Live: select sessions to include
- For Custom: select individual talent from applications
- Saves to `presentations` table with auto-generated `access_token`
- List view: name, type, session/talent count, last modified, creator
- Copy link button, activate/deactivate toggle

## Existing Infrastructure (verified)
- `presentations` table exists (migration 00010) with: `id`, `casting_call_id`, `name`, `type` (live/custom), `access_token`, `password`, `is_active`, `expires_at`, `allow_feedback`, `created_by`
- `presentation_sessions` junction table: `presentation_id`, `session_id`, `sort_order`
- `presentation_items` table: `presentation_id`, `application_id` (NOT talent_profile_id), `note`, `sort_order`
- `presentation_feedback` table exists (for future ticket)
- All TypeScript types exist: `Presentation`, `PresentationSession`, `PresentationItem`, `PresentationFeedback`
- RLS policies: "Admins manage presentations/sessions/items/feedback"
- Indexes: unique on `access_token`, index on `casting_call_id`, partial index on `is_active`
- `CastingSubNav` uses route-based navigation with `TABS` array (Overview/Applications/Requests)
- `sessions` table exists with casting sessions data
- `createClient()` from `@/lib/supabase/client` for client-side queries

## Implementation Steps

### Step 1: Add "Presentations" tab to CastingSubNav
Add `{ label: 'Presentations', segment: '/presentations' }` to the TABS array.

### Step 2: Create PresentationForm component
Props: `castingId`, `sessions` (for live type), `applications` (for custom type), `onSuccess`, `onCancel`, `presentation?` (edit mode)

Form fields:
- `name` (required text)
- `type` (select: live/custom)
- `password` (optional text)
- `expires_at` (optional date)

Conditional:
- Type=live: multi-checkbox list of sessions for this casting
- Type=custom: multi-checkbox list of applications for this casting

On submit:
1. Insert into `presentations` (access_token auto-generated by DB)
2. If live: batch insert into `presentation_sessions`
3. If custom: batch insert into `presentation_items`
4. Toast + onSuccess()

### Step 3: Create PresentationList component
Props: `presentations` (with joined data), `castingId`, `onRefresh`

Columns: Name, Type, Count, Modified, Creator, Actions

Actions per row:
- Copy Link: `navigator.clipboard.writeText(url)` → toast
- Activate/Deactivate: Switch toggle → update `is_active`
- Delete: confirmation → delete row

### Step 4: Create presentations page
`DashboardLayout` + `CastingSubNav` + breadcrumb
Fetch: presentations with joined `presentation_sessions(count)`, `presentation_items(count)`, sessions list, applications list
Render: "New Presentation" button + PresentationList + PresentationForm (in modal)

## Key Data Points
- `presentation_items.application_id` references `applications.id` (NOT talent profiles directly)
- `access_token` is auto-generated by PostgreSQL: `encode(gen_random_bytes(32), 'hex')`
- Presentation link format: `${origin}/presentations/${access_token}`
- The `allow_feedback` field exists but is for the public viewer (future ticket #62)

## Risks
| Risk | Mitigation |
|------|------------|
| Large application list in custom type selector | Limit to 100, add search/filter |
| No sessions exist for a casting | Show empty state with message to create sessions first |
| Password stored as plaintext | Acceptable for sharing passwords (not user auth) |
